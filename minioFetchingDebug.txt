The FastAPI services load environment variables from the repository's .env files
and Redis on startup. `DataStorageRetrieval.arrow_client` calls `load_dotenv` on
import so CLIENT_NAME, APP_NAME and PROJECT_NAME are available for any module
that accesses MinIO. During the FastAPI startup event we also call
`load_env_from_redis()` which populates these variables from the user's saved
environment in Redis.

To determine the current MinIO prefix you can call:

```python
from DataStorageRetrieval.arrow_client import _get_prefix
prefix = _get_prefix()  # returns "<client>/<app>/<project>/"
```

Any service may combine this prefix with the MinIO bucket name (default
"trinity") to fetch or store objects:

```python
object_name = f"{prefix}my_dataframe.arrow"
minio_client.get_object("trinity", object_name)
```
